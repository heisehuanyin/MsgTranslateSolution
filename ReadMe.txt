本项目编译的环境：
VS2017



编程语言：
Visual C++ 



目的：
本项目是希望编写一个通用的“总线消息解析平台”，所有类似于数据都可以利用此平台将抽象的数据翻译成为详尽的真实数据。
为了照顾通用化（适用平台包括但不限于1553B，TCP/IP，LVDS，PCI，I2C，串口，RS422等，众多总线通讯技术），本平台被设计成为一个基础平台加众多插件的组合平台。
基础平台提供插件中相互沟通的主要渠道，众多插件通力配合，在翻译规则文件的约束下，将总线消息翻译出来。
通过增加新插件，平台能力可以得到无限拓展，适用于各种自定义总线规则消息的解析。
暂时由于本平台尚未拓展到其他总线技术体系上，因此，软件中众多细节可能还是为1553B总线消息翻译的目的设计的，如果有人对拓展本平台到其他技术体系上感兴趣，可以自行拓展本平台。




基本工作流程：
首先，整个平台的软件运行的基本条件是拥有一个完整的parserule文件，整个文件中规定了以下要素：
	1. 解析软件运行的基础支持动态库
	2. 解析过程中用到的增强插件动态库
	3. 运行过程中用到的命令别名，以及详细命令参数
	4. 运行过程中用到的判据链
	5. 运行解析过程中详细判据以及判断过程，以及如何解析整个总线消息

以上参数都是必须的，详细解释以及格式信息可以参照随软件提供的 “示例文件.parserule”。
以上文件可以手写，也可以专门定制一个简单的前端界面来自动编写该文件。

运行软件提供的ParseCompilier程序，将parserule编译成为ParseConfigration文件，这是一个XML文件，用于在解析过程中提供判据，其中对判断过程拥有详细的规定，
结构相较于parserule文件更加严谨，但太过于繁琐，不适合人工书写。其主要信息来源于parserule文件，不详细展开。

运行本平台的解析核心MsgTranslateSolution.exe软件，将总线记录文件和ParseConfigration文件作为参数传递进去，软件会自动工作，将翻译好的数据写入指定文件。

解析好的数据可以利用Origin等软件打开，也可以利用数据库等存储，以便于开展更多数据处理操作。




项目组件：
	1.MsgTranslateSolution项目生成翻译软件主体						：平台运行的框架
	2.BinaryDataMaker项目生成模拟数据文件							：示范二进制总线数据文件
	3.其他项目生成平台用拓展插件dll									：功能插件
	4.ReadMe.txt文件												：对于整个软件的概要说明
	5.Source.parserule文件											：人类可读写的解析规则配置文件，其中附有详细的编写说明
	6.ParseConfigration.xml文件										：结构化的解析规则配置文件，便于软件高速读取和引用，繁琐，编译程序生成，编译程序位于ParseruleCompiler.MsgTranslateSolution项目
	7.程序利用dll拓展能力说明.txt									：如何利用dll对整个消息解析平台进行拓展的实用帮助
	8.导出平台的所有接口参数.bat									：利用平台提供的机制导出左右工作插件的接口参数，并提供给定输入，观察输出内容形式，便于编写parserule文件
	9.运行解析实例数据.bat											：运行软件解析解决方案提供的模拟数据，并将输出打印到屏幕上
